ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на выполнение работ по доработке информационной системы
ГБУ «Арктический научно-исследовательский центр Республики Саха (Якутия)»

1.	ОБЩИЕ ПОЛОЖЕНИЯ
1.1.	Настоящее Техническое задание определяет состав, объем, требования и результаты работ по модернизации информационной системы Заказчика (далее – Система).
1.2.	Работы выполняются в рамках 1 (одного) календарного месяца с даты подписания Договора.
1.3.	Работы выполняются в три внутренних этапа.

2.	ЭТАП № 1 – АУДИТ И ПРОЕКТИРОВАНИЕ

2.1.	Состав работ
Исполнитель обязуется:
•	провести анализ текущей архитектуры Системы;
•	провести анализ структуры сайта и контента;
•	выявить технические и логические ограничения;
•	провести базовый аудит производительности;
•	разработать архитектурную схему модернизации;
•	разработать обновленную структуру сайта (Sitemap);
•	подготовить концепцию развития CMS;
•	подготовить концепцию внедрения интеллектуального модуля (ИИ).

2.2.	Результаты этапа
•	Отчет об аудите (PDF);
•	Архитектурная схема системы;
•	Схема структуры сайта;
•	Концепция дальнейшего развития.

2.3.	Критерии приемки
•	Документы переданы Заказчику;
•	Архитектура согласована;
•	Замечания устранены (при наличии).

3.	ЭТАП № 2 – РЕОРГАНИЗАЦИЯ СТРУКТУРЫ И БАЗОВАЯ ДОРАБОТКА

3.1.	Состав работ
Исполнитель обязуется:
•	реализовать обновленную структуру сайта;
•	переработать навигацию;
•	доработать пользовательский интерфейс ключевых страниц;
•	оптимизировать скорость загрузки страниц;
•	настроить систему ролей пользователей;
•	подготовить CMS к дальнейшему расширению;
•	устранить выявленные технические ошибки.

3.2.	Технические требования
•	Время загрузки основных страниц – не более 3 секунд;
•	Отсутствие критических ошибок;
•	Корректная работа в актуальных версиях браузеров;
•	Корректное отображение на мобильных устройствах.
3.3.	Результаты этапа
•	Рабочая версия обновленного сайта;
•	Обновленная система навигации;
•	Настроенные роли пользователей.

3.4.	Критерии приемки
•	Функционал соответствует утвержденной структуре;
•	Критические ошибки отсутствуют;
•	Система функционирует стабильно.

4.	ЭТАП № 3 – ПОДГОТОВКА К ВНЕДРЕНИЮ ИНТЕЛЛЕКТУАЛЬНОГО МОДУЛЯ
В рамках настоящего этапа осуществляется подготовка инфраструктуры, без полноценного запуска ИИ-модуля.

4.1.	Состав работ
Исполнитель обязуется:
•	сформировать структуру базы знаний;
•	подготовить материалы Заказчика к структурированному хранению;
•	реализовать поисковый механизм по базе знаний;
•	подготовить API-архитектуру для будущего подключения ИИ;
•	настроить систему логирования;
•	подготовить техническую документацию для следующего этапа.

4.2.	Технические требования
•	Поиск корректно обрабатывает ключевые запросы;
•	База знаний структурирована;
•	Архитектура готова к масштабированию.

4.3.	Результаты этапа
•	Рабочий поисковый модуль;
•	Структурированная база знаний;
•	Документация на следующий этап модернизации.

4.3.	Критерии приемки
•	Поиск функционирует без критических ошибок;
•	Документация передана Заказчику;
•	Архитектура соответствует концепции развития.

5.	ОБЩИЕ ТРЕБОВАНИЯ К ВЫПОЛНЕНИЮ РАБОТ
5.1.	Работы выполняются с соблюдением требований информационной безопасности.
5.2.	Работы не должны нарушать функционирование действующих сервисов Заказчика.
5.3.	Все изменения предварительно согласовываются с Заказчиком.
5.4.	Исполнитель обеспечивает резервное копирование данных перед внесением существенных изменений.

6.	ПОКАЗАТЕЛИ ЭФФЕКТИВНОСТИ (KPI)
Работы считаются выполненными при достижении следующих показателей:
•	Выполнение объема работ по каждому этапу – не менее 100% от предусмотренного ТЗ;
•	Отсутствие критических ошибок;
•	Скорость загрузки – ≤ 3 секунд;
•	Согласование архитектурных решений Заказчиком.

7.	ИТОГОВЫЙ РЕЗУЛЬТАТ ПЕРВОГО МЕСЯЦА
По завершении работ Заказчик получает:
•	модернизированную структуру сайта;
•	обновленный интерфейс;
•	подготовленную архитектуру масштабирования;
•	основу для внедрения интеллектуального модуля;
•	техническую документацию.
